PARENT_PROGRAM_EXAMPLE_1:
```python
CONCEPT_GROUP = "combinatorics"
CONCEPT_TYPE = "combinatorics.committee_count"

"""Shallow committee-count generator (parent shape).

Single reasoning move: read off a binomial coefficient C(n, k) for
given n and k. No multi-stage counting, no inclusion-exclusion, no
constraint on roles — just one binomial evaluation.
"""

import math
import random
import sympy


def generate(seed):
    rng = random.Random(seed)
    while True:
        n = rng.randint(8, 18)
        # k strictly between 2 and n-2 so the answer is non-trivial and
        # not symmetric to a 1-element or (n-1)-element choice.
        k = rng.randint(3, n - 3)
        if k == n - k:
            continue  # avoid the symmetric midpoint where C(n,k) is "named"
        answer = math.comb(n, k)
        if answer < 20:
            continue  # too small to require real reasoning
        break
    problem = (
        f"A committee of {k} people is to be formed from a group of {n} "
        f"people. How many distinct committees are possible?"
    )
    return problem, str(sympy.Integer(answer))
```

MUTATED_PROGRAM_EXAMPLE_1:
```python
CONCEPT_GROUP = "geometry"
CONCEPT_TYPE = "geometry.power_of_point_secants"

"""In-breadth mutation child: parent worked in combinatorics
(committee count via a single binomial). This child moves to
geometry.power_of_point_secants -- a fully continuous domain with
no enumeration, no factorials, and no shared reasoning structure
with the parent.

Reasoning stages required of the solver:
  (1) recognize that for a fixed circle and an external point P, the
      product of the two intersection distances along any secant
      through P is invariant -- i.e. discover the Power of a Point
      relation PA * PB = PC * PD,
  (2) use that invariance to confirm the configuration is consistent
      (the two stated secants must yield the same product; if a
      solver verifies this, they have effectively learned the
      relation),
  (3) extend the same invariance to the tangent case, where the
      tangent length T satisfies PT^2 = PA * PB,
  (4) reduce the resulting integer PA * PB to its square-free form
      k^2 * d and report PT = k * sqrt(d).

A solver who does NOT discover the power-of-a-point relation is
forced into coordinate geometry: place P and the circle, solve for
the circle's center and radius from the four intersection
constraints, then derive the tangent length from the discriminant
of a quadratic. That route is solvable but several times longer.

Inverse construction. We sample PA, PB, PC freely and DERIVE PD
from the power-of-a-point identity (PD = PA * PB / PC). This
guarantees by construction that a consistent circle exists. We then
reject configurations where:
  - PD is non-integer (would force ugly stated lengths),
  - PD <= PC (would put D between P and C, breaking the far-point
    convention),
  - PA * PB is a perfect square (would collapse PT to an integer
    and let the solver guess by inspection),
  - the surd radicand exceeds 50 (would look artificial in the
    problem statement),
  - any two of the four stated distances coincide (creates
    accidental symmetry that may shortcut the chain).

Note on hint hygiene. The problem text does NOT name "power of a
point", does NOT mention that PA * PB equals PC * PD, and does NOT
hint that the tangent length is related to the secant products.
The solver must discover the relation.
"""

import random
import sympy


def generate(seed):
    rng = random.Random(seed)
    while True:
        PA = rng.randint(2, 8)
        # PB > PA so that B is the far intersection along the first secant.
        PB = rng.randint(PA + 4, PA + 14)
        PC = rng.randint(2, 8)
        prod = PA * PB
        # Power-of-a-point forces PD; require an integer for clean wording.
        if prod % PC != 0:
            continue
        PD = prod // PC
        # PD must be the far intersection along the second secant.
        if PD <= PC:
            continue
        # Second chord length CD = PD - PC; reject vanishingly short chords.
        if PD - PC < 3:
            continue
        # Keep all stated distances in a natural range.
        if PD > 40:
            continue

        # Reduce PT^2 = PA * PB to square-free form k^2 * d.
        d, k = prod, 1
        for q in sympy.primerange(2, 30):
            while d % (q * q) == 0:
                d //= q * q
                k *= q
        # Perfect square -> integer tangent -> solver guesses by inspection.
        if d == 1:
            continue
        # Surd radicand must stay small and clean to state.
        if d > 50:
            continue

        # Degeneracy guards: distinct stated distances avoid accidental
        # symmetric shortcuts.
        if PA == PC or PB == PD:
            continue
        if len({PA, PB, PC, PD}) < 4:
            continue

        answer = f"sqrt({d})" if k == 1 else f"{k}*sqrt({d})"
        break

    problem = (
        f"From a point P outside a circle, two straight lines are drawn "
        f"that each cross the circle in two places. The first line meets "
        f"the circle at points A and B with PA = {PA} and PB = {PB}. "
        f"The second line meets the circle at points C and D with "
        f"PC = {PC} and PD = {PD}. A third line from P is drawn so that "
        f"it just touches the circle at a single point T. Compute the "
        f"length PT in exact form."
    )
    return problem, answer
```
