PARENT_A_PROGRAM_EXAMPLE_1:
```python
CONCEPT_GROUP = "combinatorics"
CONCEPT_TYPE = "combinatorics.binomial_mod_lucas"

"""Shallow committee-count generator (parent shape A).

Single reasoning move: read off a binomial coefficient C(n, k) for
given n and k. No multi-stage counting, no inclusion-exclusion, no
modular reduction — just one binomial evaluation.
"""

import math
import random
import sympy


def generate(seed):
    rng = random.Random(seed)
    while True:
        n = rng.randint(8, 18)
        # k strictly between 2 and n-2 so the answer is non-trivial and
        # not symmetric to a 1-element or (n-1)-element choice.
        k = rng.randint(3, n - 3)
        if k == n - k:
            continue  # avoid the symmetric midpoint where C(n,k) is "named"
        answer = math.comb(n, k)
        if answer < 20:
            continue  # too small to require real reasoning
        break
    problem = (
        f"A committee of {k} people is to be formed from a group of {n} "
        f"people. How many distinct committees are possible?"
    )
    return problem, str(sympy.Integer(answer))
```

PARENT_B_PROGRAM_EXAMPLE_1:
```python
CONCEPT_GROUP = "number_theory"
CONCEPT_TYPE = "number_theory.modular_reduction"

"""Shallow modular-residue generator (parent shape B).

Single reasoning move: reduce a stated integer modulo a small prime
by direct division. No identities, no digit decomposition, no
chaining — just one modular reduction.

The point of this parent is to introduce the small-prime modular
arithmetic primitive in its plainest form, so that a downstream
crossover can recognise it as one of the two ingredients to fuse.
"""

import random
import sympy


def generate(seed):
    rng = random.Random(seed)
    while True:
        # Small odd prime — keeps the residue compact and the verifier fast.
        p = rng.choice([5, 7, 11, 13, 17, 19])
        # Integer large enough that the residue is not visible by inspection
        # (no "30 mod 7" trivialities) but not so large that the problem
        # looks artificial as a standalone.
        N = rng.randint(200, 9_000)
        r = N % p
        # Reject named-trivial residues so the parent's single move still
        # carries a non-zero amount of work.
        if r in (0, 1, p - 1):
            continue
        # Reject the case where N itself ends in a way that gives away r
        # by the last digit (e.g. p=5 with N ending in 3 → r=3 obvious).
        if p == 5 and N % 10 == r:
            continue
        if p == 2:  # excluded by the prime pool, but defensive
            continue
        break
    problem = (
        f"Compute the remainder when {N} is divided by {p}. "
        f"State only the remainder as an integer between 0 and {p - 1}."
    )
    return problem, str(sympy.Integer(r))
```

CROSSOVER_CHILD_PROGRAM_EXAMPLE_1:
```python
CONCEPT_GROUP = "combinatorics"
CONCEPT_TYPE = "combinatorics.committee_count"

"""Crossover hybrid generator.

Parent A (combinatorics.binomial_mod_lucas):
    The binomial coefficient C(n, k) counts the number of k-element
    subsets of an n-element set — the canonical committee-count
    object.

Parent B (number_theory.modular_reduction):
    Reductions modulo a small prime p decompose large integers into
    their p-adic digits, turning operations on giant numbers into
    operations on small ones.

Shared structure: the single quantity C(n, k) mod p plays BOTH
roles simultaneously. Combinatorially it is the (residue of the)
count of k-subsets; arithmetically it factors digit-by-digit via
Lucas's theorem,
    C(n, k)  ≡  ∏ C(n_i, k_i)   (mod p),
where n = Σ n_i p^i and k = Σ k_i p^i are the base-p expansions.
The two parents are not concatenated; the same object C(n, k) mod p
is acted on by the combinatorial identity AND the number-theoretic
digit reduction at once.

Reasoning stages required of the solver:
  (1) recognise that direct evaluation of C(n, k) is hopeless for
      the stated n, k (the integer has thousands of digits),
  (2) recall / discover Lucas's theorem to reduce the mod-p value
      to a product of single-digit binomials,
  (3) expand both n and k in base p and pair the digits,
  (4) evaluate each small C(n_i, k_i), multiply, and reduce mod p.

A solver who does NOT find Lucas is stuck: factorials of multi-
thousand integers are beyond direct manipulation, and no shorter
combinatorial argument is available.

Inverse construction. We sample p in {5, 7, 11, 13} and n in
[p^3, p^4 - 1], so n has 3-4 base-p digits — too many for direct
factorial expansion but few enough that Lucas's product has 3-4
small factors. k is sampled in a non-trivial band of n.

Validity guards (rejection sampling):
  - any digit pair (n_i, k_i) with k_i > n_i makes Lucas yield 0,
    which collapses the answer to a trivial residue -> reject;
  - fewer than two "non-trivial" digit pairs (with 0 < k_i < n_i)
    means the product reduces to a single binomial, defeating the
    crossover -> reject;
  - answers in {0, 1, p - 1} let the solver guess by inspection
    -> reject;
  - 2k = n exactly creates an accidental symmetry shortcut -> reject.

Cross-check. We assert that the Lucas digit-wise product matches
direct sympy.binomial(n, k) % p. The assertion is a development-
time guard; in deployed runs it is never triggered, but keeps the
generator honest if any future edit perturbs the digit logic.

Note on hint hygiene. The problem text states only n, k, p and
asks for the residue. It does NOT mention Lucas's theorem, does
NOT mention p-adic expansion, and does NOT factor n or k for the
solver.
"""

import random
import sympy
from sympy import Integer, binomial


def _p_adic(n, p):
    """Return [d0, d1, d2, ...] with n = sum d_i * p^i, 0 <= d_i < p."""
    digits = []
    while n > 0:
        digits.append(n % p)
        n //= p
    return digits


def generate(seed):
    rng = random.Random(seed)
    while True:
        p = rng.choice([5, 7, 11, 13])

        # n in [p^3, p^4 - 1] so direct factorial evaluation is hopeless
        # for the solver but Lucas reduces to <= 4 single-digit binomials.
        n = rng.randint(p ** 3, p ** 4 - 1)
        k = rng.randint(n // 4, 3 * n // 4)

        if k == 0 or k == n:
            continue
        if 2 * k == n:
            continue  # accidental symmetric midpoint

        n_digits = _p_adic(n, p)
        k_digits = _p_adic(k, p)
        while len(k_digits) < len(n_digits):
            k_digits.append(0)

        # Reject digits where Lucas gives 0 (trivial answer).
        if any(kd > nd for nd, kd in zip(n_digits, k_digits)):
            continue

        # Require at least two non-trivial digit pairs so the Lucas
        # product is a genuine multi-stage computation, not one
        # disguised binomial.
        nontrivial = sum(
            1 for nd, kd in zip(n_digits, k_digits) if 0 < kd < nd
        )
        if nontrivial < 2:
            continue

        # Lucas digit-wise product.
        answer = 1
        for nd, kd in zip(n_digits, k_digits):
            answer = (answer * int(binomial(nd, kd))) % p

        if answer in (0, 1, p - 1):
            continue

        actual = int(binomial(n, k)) % p
        assert actual == answer

        break

    problem = (
        f"Compute the remainder when the binomial coefficient "
        f"C({n}, {k}) = {n}! / ({k}! * {n - k}!) is divided by {p}. "
        f"State only the remainder as an integer between 0 and {p - 1}."
    )
    return problem, str(Integer(answer))
```
