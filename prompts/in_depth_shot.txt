MUTATED_PROGRAM_EXAMPLE_1:
```python
"""In-depth mutation generator.

Deepens modular_reduction: the parent reduces a given integer
N mod p in one step; this child asks for a^b mod p with b far
too large to form a^b directly.

To solve, given only a, b, p, the reader must:
  (1) see that forming a^b directly is hopeless,
  (2) recall Fermat's little theorem a^(p-1) ≡ 1 (mod p) for
      gcd(a, p) = 1,
  (3) reduce the exponent to b' = b mod (p-1),
  (4) reduce a^b' mod p — the parent's original move.
Fast modular exponentiation also works but costs several times
more multiplications and effectively rediscovers FLT along the
way.

Guards keep the task honest: b' not in {0, 1} (else it collapses
to 1 or a mod p), final residue not in {0, 1, p-1} (guessable),
a coprime to p (FLT precondition, forced by 2 <= a < p prime).
The problem text states only a, b, p — never FLT or the exponent
reduction.
"""

import random
import sympy

def generate(seed):
    rng = random.Random(seed)
    while True:
        p = rng.choice([7, 11, 13, 17, 19, 23])
        a = rng.randint(2, p - 1)
        b = rng.randint(p ** 2, p ** 3)

        b_reduced = b % (p - 1)
        if b_reduced in (0, 1):
            continue

        answer = pow(a, b, p)
        if answer in (0, 1, p - 1):
            continue
        break

    problem = (
        f"Compute the remainder when {a}^{b} is divided by {p}. "
        f"State only the remainder as an integer between 0 and {p - 1}."
    )
    return problem, str(sympy.Integer(answer))

CONCEPT_REASON = "To solve, the reader must recognise a too-large exponent forces Fermat's little theorem to reduce it mod (p-1) before the final mod-p reduction — that exponent reduction is the problem's reasoning core."
CONCEPT_GROUP = "number_theory"
CONCEPT_TYPE = "number_theory.modular_reduction"
```

PARENT_PROGRAM_EXAMPLE_2:
```python
import random
import sympy

def generate(seed):
    rng = random.Random(seed)
    while True:
        x = sympy.symbols('x')
        deg = rng.randint(2, 3)
        coeffs = [rng.randint(-5, 5) for _ in range(deg + 1)]
        if coeffs[0] == 0:
            continue
        c = rng.randint(-4, 4)
        if c == 0:
            continue
        poly = sum(coeffs[i] * x ** (deg - i) for i in range(deg + 1))
        answer = int(poly.subs(x, c))
        if abs(answer) < 5:
            continue
        break
    terms = []
    for i, k in enumerate(coeffs):
        power = deg - i
        if k == 0:
            continue
        if power == 0:
            terms.append(f"{k:+d}")
        elif power == 1:
            terms.append(f"{k:+d}*x")
        else:
            terms.append(f"{k:+d}*x^{power}")
    expr = " ".join(terms).lstrip("+ ").replace("+ -", "- ")
    problem = (
        f"Let p(x) = {expr}. Compute p({c})."
    )
    return problem, str(sympy.Integer(answer))

CONCEPT_GROUP = "algebra"
CONCEPT_TYPE = "algebra.polynomial_evaluation"
```

MUTATED_PROGRAM_EXAMPLE_2:
```python
"""In-depth mutation generator.

Deepens polynomial_evaluation: the parent substitutes a point
into a small-degree polynomial; this child asks for the
remainder of a higher-degree p(x) divided by (x - c), the
linear divisor lightly disguised so the identity is not a
surface match.

To solve, given the polynomial and the divisor, the reader
must:
  (1) see that explicit polynomial division is messy at the
      stated degree,
  (2) recall the remainder theorem p(x) mod (x - c) = p(c),
  (3) read c off the divisor,
  (4) evaluate p(c) — the parent's single-point substitution.
Synthetic division is the only short alternative, and is
itself a disguised execution of the remainder theorem.

Guards keep the task honest: leading coefficient != 0 (degree
preserved), |c| <= 4 (powers do not blow into an
arithmetic-stamina test), final value not in {-1, 0, 1}
(guessable). The problem text states only the polynomial and
the divisor (x - c) — never the remainder theorem or the
substitution shortcut.
"""

import random
import sympy

def generate(seed):
    rng = random.Random(seed)
    x = sympy.symbols('x')

    while True:
        deg = rng.randint(5, 7)
        coeffs = [rng.randint(-4, 4) for _ in range(deg + 1)]
        if coeffs[0] == 0:
            continue

        c = rng.randint(-3, 3)
        if c == 0:
            continue

        poly = sum(coeffs[i] * x ** (deg - i) for i in range(deg + 1))
        answer = int(poly.subs(x, c))

        if answer in (-1, 0, 1):
            continue
        if abs(answer) > 500:
            continue
        break

    terms = []
    for i, k in enumerate(coeffs):
        power = deg - i
        if k == 0:
            continue
        if power == 0:
            terms.append(f"{k:+d}")
        elif power == 1:
            terms.append(f"{k:+d}*x")
        else:
            terms.append(f"{k:+d}*x^{power}")
    expr = " ".join(terms).lstrip("+ ").replace("+ -", "- ")

    divisor = f"x - {c}" if c > 0 else f"x + {-c}"

    problem = (
        f"Let p(x) = {expr}. Compute the remainder when p(x) is "
        f"divided by ({divisor}). State only the remainder as an "
        f"integer."
    )
    return problem, str(sympy.Integer(answer))

CONCEPT_REASON = "To solve, the reader must recognise division by a linear factor (x - c) reduces to evaluating p(c) via the remainder theorem — that substitution is the problem's reasoning core."
CONCEPT_GROUP = "algebra"
CONCEPT_TYPE = "algebra.polynomial_evaluation"
```
